What it does

1

Scan

Detects OS, GPU, RAM, disk, runs speed test. Scans 14 tools across dev, network, AI, and local AI. Reports installed vs missing.

2

Choose

Pick a preset (Minimal dev, Developer, AI workstation) or Custom. Presets skip all questions. Custom lets you pick everything yourself.

3

Authenticate

Interactive login for GitHub CLI, Tailscale, Cloudflare. Done early so the rest runs unattended.

4

Configure

Sets git identity, restores SSH key from vault or generates new one (adds to GitHub), applies dev settings, saves config.

5

Install

Fully unattended. Installs via winget/brew/apt, npm, and ollama pull. Progress saved for resume on failure. Plus VS Code extensions and dotfiles.

6

Clone repos

Reads config/repos.json and clones each repo. Private repos that 404 are skipped with a warning.

7

Verify

Checks every tool on PATH with version output. Functional tests: Python imports, Node.js, Ollama inference timing. Reports total elapsed time.

Quick start

Clone this repo: git clone https://github.com/jeremytrindade/jet-init.git

Windows: double-click jet-init.bat. macOS/Linux: run ./jet-init.sh

Pick a preset (Minimal dev, Developer, AI workstation) or choose Custom.

Answer auth questions (~2 min). Everything after this runs unattended.

Walk away. Come back to a fully configured PC.

v1.2 features

Cross-platform

Windows (winget), macOS (Homebrew), Linux (apt). Single config, three runners.

Preset profiles

Minimal dev, Developer, or AI workstation. One choice replaces 8 questions.

Dotfiles management

Clone your dotfiles repo, symlink/copy to the right places. Configure in config/dotfiles.json.

Vault integration

Bitwarden CLI restores SSH keys from your vault. Falls back to generating new keys.

Smart model picks

Scores all AI models against your VRAM, RAM, and disk. Suggests the 3 best-fit models.

Functional tests

Phase 7 runs Python import test, Node.js test, Ollama inference with timing, git identity check.

Hardware detection

GPU (nvidia-smi, WMI, system_profiler), RAM, disk. Clear verdict for local AI capability.

SSH key

Generates ed25519 key and adds it to GitHub via gh ssh-key add.

Resume on failure

Progress saved after each install. Re-run to pick up where it left off.

VS Code extensions

Auto-installs extensions from config/vscode-extensions.json.

Install scope

Windows: current user only or all users (winget --scope user vs machine).

Speed test

Measures download speed via Cloudflare CDN for accurate time estimates.

Update mode

-Update / --update upgrades all installed tools to latest.

Dry run

-DryRun / --dry-run shows what would happen without changing anything.

Log file

All output saved to jet-init-YYYY-MM-DD.log automatically.

Customize for your own setup

Fork this repo and edit the config/ folder. The scripts read everything from config, so you never need to touch the engine code.

config/defaults.json

Your workspace path, GitHub username, git email. The single identity file.

config/repos.json

Your repos to clone. Mark required repos, set shallow clone for large/optional ones.

config/vscode-extensions.json

Your VS Code extensions to auto-install.

config/dotfiles.json

Your dotfiles repo URL and file-to-path mappings. Leave repo empty to skip.

config/catalog.json

Add or remove tools. Each tool has install methods per OS (winget, brew, apt, npm, script).

How it works

Two entry points, same flow. Windows uses jet-init.bat + PowerShell. macOS/Linux uses jet-init.sh + bash. Both read the same config/ files.

Windows: jet-init.bat [-Update] [-DryRun]
  -> pwsh/powershell -ExecutionPolicy Bypass -File jet-init.ps1

macOS/Linux: ./jet-init.sh [--update] [--dry-run]
  -> bash + python3 for JSON parsing

Both run the same phases:
  -> Phase 1: Scan (hardware + speed test + tools)
  -> Phase 2: Choose (preset or custom)
  -> Phase 3: Authenticate (gh, tailscale, cloudflared)
  -> Phase 4: Configure (git, SSH key/vault, dev settings)
  ---- no more user input after this ----
  -> Phase 5: Install (winget/brew/apt + npm + ollama + extensions)
  -> Phase 5.5: Dotfiles (clone repo + symlink/copy)
  -> Phase 6: Clone repos (from config/repos.json)
  -> Phase 7: Verify (PATH check + functional tests + timing)
